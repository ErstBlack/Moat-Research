## Cluster compression — does it survive n=9?

The prior calibration (n=4) flagged compression and deferred edits at n≥6. At n=9:

- **Range 1.197 / σ 0.384 is a normal distribution shape, not pathological.** Random scoring on a [5, 10] sub-criterion range with the multiplicative formula will compress — that's the formula working as designed (the geometric weighting over similarly-sized inputs always produces a similarly-sized output).
- **Ordering is sensible.** NOTAMs (top) is clearly stronger than transit (bottom) on every axis. No counter-intuitive rankings.
- **The closest pair is FERC 7.006 / CSLB 7.009 — Δ=0.003.** Examining: CSLB has weaker financial (6.333 vs FERC 6.833) but stronger implementation (7.500 vs 7.000) and hardware (7.500 vs 7.250). The multiplicative formula treats those trade-offs as roughly equivalent, which is a defensible design choice — both briefs would reach the same operator decision (both currently scored, neither approved).
- **No decision-quality failure.** Composite is one input the operator weighs against the prose moat-thesis when actually approving a brief. The compression is not preventing decisions.

## Recommendation: **defer formula edits, document trigger conditions**

Rationale:

1. **Comparability cost is high.** The graduated brief `07.221-somd-cameras` is anchored in the current formula. Any weight tweak ripples through every existing score and breaks the historical comparability that the calibration baseline depends on.
2. **No decision-quality failure observed.** Ordering is intuitive; the only sub-0.05 pair (FERC/CSLB) doesn't drive different operator decisions.
3. **Compression is a feature, not a bug.** The geometric formula's "any-axis-zero kills" property is what enforces §5 — switching to arithmetic mean would break that property. Within the working range, similarly-shaped inputs producing similarly-shaped outputs is normal.
4. **Operator-policy guidance is bias-toward-defer.** Per planner notes ("avoid premature formula edits") and per FOCUS.md task body.

**Trigger conditions for revisiting (file a new calibration pass if any fire):**

- Any approved-vs-rejected operator decision conflict where two briefs differ by <0.10 on composite but the operator splits them on prose-moat reasoning the rubric doesn't capture.
- A new brief whose composite ranks counter-intuitively against an experienced reader's first-pass judgement (sigma > 1 from operator expectation).
- n grows past 15 with the L1↔L4 boundary blurring (e.g., L4 brief composite > 7.500 with all-publicly-archived inputs and weak compute-barrier evidence).
- A new brief that is hardware-9 but defensibility-3 reaches a top-quartile composite. (Currently the rubric handles this via the §5 hard floor; trigger is if a near-floor case slips through.)

Of the four formula alternatives sketched (weight 0.5/0.25/0.25, arithmetic mean, defensibility-2x-weighted financial sub-avg, sub-criterion full-range guidance), only the **sub-criterion full-range scoring guidance** (Option F — anchor explicit "1=bottom-of-market, 5=median, 10=best-in-class" descriptors) is a low-risk pre-edit candidate worth keeping in the back pocket. It's documentation-only, doesn't break comparability, and would let future scorers stretch the [5, 10] band into [3, 10]. **Not adopting it now** — operator hasn't indicated the band is a problem, and adding scoring-guidance anchors is its own multi-iteration effort to do well.
